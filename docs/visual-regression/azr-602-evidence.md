# AZR-602 Visual Regression Evidence

Date: 2026-06-07

## Optimization Gate

- Bottleneck targeted: visual regressions and frame-time regressions were only
  caught by ad hoc manual browser checks, so black screens, color drift, missing
  sprites, or slow direct-file rendering could ship without a repeatable gate.
- Representation/lifecycle boundary changed: rendered WebGL2 frames are now
  captured as deterministic PNG baselines during dev verification. The runtime
  remains unchanged and still loads through vanilla script tags.
- Chunk, batch, or aggregate boundary: each capture uses one deterministic
  watcher view state: orbit, temperate surface, desert surface, entity-visible
  surface, and HUD-visible view.
- Readiness state required: captures wait for `PS.render.webgl2Renderer.gl`
  before screenshots are consumed, then force deterministic world/camera state
  and redraw.
- Player-perception contract preserved: the gate covers direct `file://`
  rendering, visible planet pixels, biome readability, entity visibility, HUD
  presence, no debug errors, and frame timing under the AZR-602 budget.
- New constraint or encoding limit: the PNG diff decoder supports 8-bit,
  non-interlaced RGB/RGBA PNG screenshots generated by Playwright/Chromium.
- Metric proving the bottleneck moved: `npm run test:visual` fails when more
  than 5% of pixels change from the golden baseline and asserts average frame
  time `< 20ms` plus peak frame time `< 50ms` during a 100-frame browser pass.
